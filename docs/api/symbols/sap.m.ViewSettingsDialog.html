Class sap.m.ViewSettingsDialog

extends Control

ViewSettingsDialog provides functionality to easily select the options for sorting, grouping, and filtering data. It is a composite control, consisting of a modal popover and several internal lists. There are three different tabs (Sort, Group, Filter) in the dialog that can be activated by filling the respecive associations. If only one assication is filled, the other tabs are automatically hidden. The selected options can be used to create sorters and filters for the table.


Since: 1.16.

new sap.m.ViewSettingsDialog(sId?, mSettings?)
Constructor for a new ViewSettingsDialog.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers.

If the name of a setting is ambiguous (e.g. a property has the same name as an event), then the framework assumes property, aggregation, association, event in that order. To override this automatic resolution, one of the prefixes "aggregation:", "association:" or "event:" can be added to the name of the setting (such a prefixed name must be enclosed in single or double quotes).

The supported settings are:

  • Properties
    • title : string
    • sortDescending : boolean (default: false)
    • groupDescending : boolean (default: false)
  • Aggregations
    • sortItems : sap.m.ViewSettingsItem[]
    • groupItems : sap.m.ViewSettingsItem[]
    • filterItems : sap.m.ViewSettingsItem[]
    • presetFilterItems : sap.m.ViewSettingsItem[]
  • Associations
    • selectedSortItem : string | sap.m.ViewSettingsItem
    • selectedGroupItem : string | sap.m.ViewSettingsItem
    • selectedPresetFilterItem : string | sap.m.ViewSettingsItem
  • Events
    • confirm : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • cancel : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • resetFilters : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
Parameters:
{string} sId? id for the new control, generated automatically if no id is given
{object} mSettings? initial settings for the new control

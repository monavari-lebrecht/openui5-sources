new sap.ui.core.delegate.ScrollEnablement(oControl, sScrollContentDom, oConfig)
Creates a ScrollEnablement delegate that can be attached to Controls requiring capabilities for scrolling of a certain part of their DOM on mobile devices.
Parameters:
{sap.ui.core.Control} oControl the Control of which this Scroller is the delegate
{string} sScrollContentDom the Id of the element within the DOM of the Control which should be scrollable
{object} oConfig the configuration of the scroll delegate
{boolean} oConfig.horizontal?, Default: false Whether the element should be scrollable horizontally
{boolean} oConfig.vertical?, Default: false Whether the element should be scrollable vertically
{boolean} oConfig.zynga?, Default: false If set, then the Zynga scroller (http://zynga.github.com/scroller/) is used
{boolean} oConfig.iscroll?, Default: false If set, then iScroll (http://cubiq.org/iscroll-4) is used
{boolean} oConfig.preventDefault?, Default: false If set, the default of touchmove is prevented
{boolean} oConfig.nonTouchScrolling?, Default: false If true, the delegate will also be active to allow touch like scrolling with the mouse on non-touch platforms; if set to "scrollbar", there will be normal scrolling with scrollbars and no touch-like scrolling where the content is dragged

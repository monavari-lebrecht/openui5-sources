Class sap.ui.commons.AccordionSection

extends Element

Represents a panel which is a container for other controls. The container does not have any layout function.

Constructor Detail
new sap.ui.commons.AccordionSection(sId?, mSettings?)
Constructor for a new AccordionSection.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers.

If the name of a setting is ambiguous (e.g. a property has the same name as an event), then the framework assumes property, aggregation, association, event in that order. To override this automatic resolution, one of the prefixes "aggregation:", "association:" or "event:" can be added to the name of the setting (such a prefixed name must be enclosed in single or double quotes).

The supported settings are:

  • Properties
    • maxHeight : sap.ui.core.CSSSize
    • enabled : boolean (default: true)
    • collapsed : boolean (default: false)
    • title : string
  • Aggregations
    • content (default aggregation) : sap.ui.core.Control[]
  • Associations
    • Events
      • scroll : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

    In addition, all settings applicable to the base type sap.ui.core.Element can be used as well.

    Parameters:
    {string} sId? id for the new control, generated automatically if no id is given
    {object} mSettings? initial settings for the new control

    Getter for aggregation content.
    Aggregates the controls that are contained in the panel. Control layouting is browser-dependent. For a stable content layout, use a layout control as direct single child. When the panel dimensions are set, the child control may have width and height of 100%; when the panel dimensions are not set, the child defines the panel size.

    Note: this is the default aggregation for AccordionSection.

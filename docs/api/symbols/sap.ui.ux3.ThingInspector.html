new sap.ui.ux3.ThingInspector(sId?, mSettings?)
Constructor for a new ThingInspector.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers.

If the name of a setting is ambiguous (e.g. a property has the same name as an event), then the framework assumes property, aggregation, association, event in that order. To override this automatic resolution, one of the prefixes "aggregation:", "association:" or "event:" can be added to the name of the setting (such a prefixed name must be enclosed in single or double quotes).

The supported settings are:

  • Properties
    • firstTitle : string
    • type : string
    • icon : sap.ui.core.URI
    • secondTitle : string
    • followState : sap.ui.ux3.FollowActionState (default: sap.ui.ux3.FollowActionState.Default)
    • flagState : boolean (default: false)
    • favoriteState : boolean (default: false)
    • favoriteActionEnabled : boolean (default: true)
    • updateActionEnabled : boolean (default: true)
    • followActionEnabled : boolean (default: true)
    • flagActionEnabled : boolean (default: true)
    • headerType : sap.ui.ux3.ThingViewerHeaderType (default: sap.ui.ux3.ThingViewerHeaderType.Standard)
  • Aggregations
    • actions : sap.ui.ux3.ThingAction[]
    • headerContent : sap.ui.ux3.ThingGroup[]
    • facets : sap.ui.ux3.NavigationItem[]
    • facetContent : sap.ui.ux3.ThingGroup[]
    • actionBar : sap.ui.ux3.ActionBar
  • Associations
    • selectedFacet : string | sap.ui.ux3.NavigationItem
  • Events
    • actionSelected : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • facetSelected : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • feedSubmit : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

In addition, all settings applicable to the base type sap.ui.ux3.Overlay can be used as well.

Parameters:
{string} sId? id for the new control, generated automatically if no id is given
{object} mSettings? initial settings for the new control

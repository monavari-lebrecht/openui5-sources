SAPUI5 Theming Concept: Right-to-left Support

In the SAPUI5 theming concept, RTL support is defined as follows:

  • All controls are styled for LTR in their usual CSS files.

  • Images are stored directly in the img subfolder or in a folder in the img subfolder.

  • The CSS generator provides an RTL flipping algorithm that performs the following operations:

    border-left: is converted to border-right:, padding: 1px 2px 3px 4px; is converted to padding: 1px 4px 3px 2px;, float:left; is converted to float:right;, and so on. By this, most tasks for RTL conversion are covered. The algorithm supports most CSS properties, including the CSS3 properties.

  • The CSS generator also mirrors all images that are stored in the img folder.

    For images that must not be mirrored in RTL mode, the RTL version must be stored in the img-RTL folder with the same location and name. For these images, the image reference in the CSS is modified to point to this image.

  • Controls shall not write special CSS classes to notify the CSS that RTL is switched on. In CSS the dir attribute of the <HTML> tag can be checked to provide a specific style that has to be applied for RTL.

    Note The written style is still mirrored in the actual RTL case.

    Example: 

    	html[dir=rtl] .sapUiBtn {
	   color: red;
	}

    However, this should only be required in rare cases. If it is used too often, check the automatic CSS mirroring functions provided in SAPUI5.

Renderer Methods

The renderer method is responsible for creating the HTML structure for the control. The renderer method is a static method, so no this keyword is available, but a control instance and a RenderManager instance are given to the method. The RenderManager collects and concatenates string fragments and places them in the DOM at the appropriate position.

renderer: function(rm, oControl) {
   oRenderManager.write("<div>", oControl.getText(), "</div>");
}

All the features and rules for writing controls also apply to the renderer control. For example, it must only have one HTML element as root node. oRenderManager.writeControlData(oControl); must be called in the root node. Thus, the root element can be marked as SAPUI5 control root and the ID of the control is written.

A new renderer type inherits from the renderer of the parent control. If a control extends, for example, the TextField control, the function is added to a class that inherits from sap.ui.commons.!TextFieldRenderer and can access the respective functions.

If an existing renderer is used without modification, you can use the name of the respective renderer class: 

renderer: "sap.ui.commons.ButtonRenderer"

A control renderer can also override or implement methods from the renderer superclass. And it can just separate out helper functions.

This is also possible in the following example, but in this case the methods need to be packed together into an object to indicate for the extend method that they all go into the control renderer. The main rendering method is called render just like in a normal control renderer. The this keyword refers to the control renderer type and is used to access the other methods: 

renderer: {

   render: function(rm, oControl) {

      rm.write("<div>");

      rm.writeEscaped(this.square(oControl.getValue()));

      rm.write("</div>");

   },

   square: function(value) {

      return value * value;

   }

}

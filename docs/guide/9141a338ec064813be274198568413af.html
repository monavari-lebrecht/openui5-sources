Resource Handling

There are two layers in SAPUI5 that are responsible for resource handling and that work completely independently from each other: the client side and the server side.

The resource handling of SAPUI5 is separated in two parts - a client-side and a server-side resource handling.

Server-side Resource Handling

SAPUI5 provides resource handlers for different servers. The main purpose of server-side resource handlers is to provide access to the SAPUI5 libraries. It improves the interaction between client and server, for example in providing a server-side locale fallback for the language, which helps to avoid multiple requests to get the correct language. It is also used to support modularized development of SAPUI5 applications and libraries.The Java resource handler is aligned with the concept of the JavaServer Faces.

Client-side Resource Handling

On the client-side, SAPUI5 provides the following mechanisms to manage resources:

  • It supports localization of application texts with resource bundles.

  • It allows JavaScript files to be modularized.

Both concepts load additional resources from a server that can be any kind of web server. It does not depend on any server.side technology.
